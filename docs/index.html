--- layout: default title: BindCaptain nav_order: 1 --- # BindCaptain Documentation Welcome to the complete BindCaptain documentation. This guide covers everything you need to know to effectively deploy and manage containerized BIND DNS servers. ## Quick Start Deploy a containerized BIND DNS server in just 3 steps: 1. **Clone & Setup** - Clone the repository and auto-install dependencies 2. **Configure DNS** - Use interactive wizard or manual setup 3. **Launch & Test** - Build and run your DNS container [Get Started →](installation.md) ## What is BindCaptain? BindCaptain is a modern, containerized BIND DNS solution that provides: - **Deploy in Minutes** - 3-step setup process - **Containerized** - Clean, isolated BIND installation - **Smart Management** - CLI tools for DNS record management - **Secure** - Modern BIND 9.16+ with security best practices - **Auto-Reverse DNS** - Automatic reverse DNS generation - **Auto-Updates** - Built-in git refresh functionality - **Production-Ready** - Used in real production environments ## Documentation ### Getting Started - [Installation Guide](installation.md) - Complete setup and configuration - [Quick Start Guide](quick-start.md) - Get up and running in minutes - [System Requirements](system-requirements.md) - Supported distributions and prerequisites ### User Guides - [DNS Operations](dns-operations.md) - Managing DNS records and zones - [Configuration Management](configuration.md) - Customizing your DNS setup - [Systemd Service](systemd-service.md) - Service management guide ### Reference - [Cheat Sheet](cheat-sheet.md) - Quick command reference - [Troubleshooting](troubleshooting.md) - Common issues and solutions ## Getting Help If you can't find what you're looking for in this documentation: 1.
